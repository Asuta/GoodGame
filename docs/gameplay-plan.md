# GoodGame Editor 玩法迭代建议

## 目标

当前项目已经具备完整的基础循环：

- 序章
- 每日行动
- 属性变化
- 条件事件
- 对话分支
- 进入下一天

现阶段的重点不再是单纯增加编辑功能，而是增强玩法上的决策张力、阶段推进感和重复游玩的动力。

## 当前玩法判断

项目现在的核心循环已经成立，但更偏“可试玩的原型”，距离“有明显可玩性和重复体验”的状态，还差以下几点：

- 各属性的职责还不够清晰
- 行动大多偏正收益，取舍感不强
- 事件更多是阈值奖励，缺少连续后果
- 缺乏中期目标和路线分化
- 缺少会影响后续体验的状态层

简单说，现在的问题不是不能玩，而是玩家还不够容易产生“这一天我要认真选”的感觉。

## 优先建议

### 1. 明确每个属性的玩法职责

不要让所有属性都只是“越高越好”，建议给它们不同的系统意义：

- `信赖`：解锁更私密的剧情、分支和关键事件
- `健康`：决定是否能执行高消耗行动，过低时限制玩法
- `心情`：影响当天行动效果或分支成功表现
- `学识`：解锁理解过去、阅读、成长向事件

这样玩家面对的不是单纯堆数值，而是在选择养成方向。

### 2. 增加阶段目标

建议加入每 5 天或 7 天一个阶段目标，让玩家知道短期目标是什么，例如：

- 让她愿意主动开口
- 陪她完成第一次出门
- 让她开始稳定阅读

阶段目标达成后可解锁：

- 新场景
- 新行动
- 新事件池
- 新对话风格

这样日常循环会有推进感，不只是机械地过天数。

### 3. 强化行动取舍

目前最值得优先优化的是“行动代价”。建议让行动之间出现真实权衡：

- `外出散步`：提升信赖，但消耗更多体力或降低健康
- `学习识字`：提升学识，但降低心情
- `休息`：提升健康，但可能错过部分事件
- `追问过去`：高信赖时收益高，低信赖时有明显负反馈

只要选择开始带来代价，玩法就会立刻更有思考空间。

### 4. 引入关系阶段

建议不要只依赖单一 `trust >= X` 阈值，而是加入更具表现力的关系阶段：

- 戒备
- 试探
- 依赖
- 主动靠近

关系阶段变化后，可以同步影响：

- 可触发事件
- 可选行动
- 对话文本风格
- 立绘、表情或头像表现

这会让“关系真的在变化”的感受比单纯数值变化更明显。

### 5. 限制刷同一行动的最优解

为了避免玩家反复刷一个最优选项，建议加入连续行为后果：

- 连续做同一种行动时收益递减
- 连续外出会积累疲劳
- 连续休息会拖慢主线推进
- 连续学习可能让心情下降更明显

这样系统会自然鼓励多样化决策。

### 6. 让事件形成链式后果

目前事件更像独立触发点，建议逐步改成事件链：

- A 事件触发后，解锁 B、屏蔽 C
- 某次分支失败，不只是掉数值，还会留下后续影响
- 后续事件根据之前的选择改写台词、结果或路线

这样玩家会开始重视“我之前做过什么”，叙事厚度会明显提升。

## 适合当前题材的增强方向

### 1. 时间段系统

把“一天 3 点行动力”进一步升级成“早 / 午 / 晚”的时间段结构。

好处：

- 某些事件只在特定时段触发
- 某些行动在不同时间段收益不同
- 生活感会明显增强

这很适合当前的陪伴 / 寄宿题材。

### 2. 状态系统

建议增加持续 1 到 3 天的短期状态，而不是只看长期属性，例如：

- 感冒
- 焦虑
- 愿意交流
- 抵触外出

状态可以影响：

- 行动是否可执行
- 行动效果强弱
- 可触发事件
- 对话风格

这样每天的体验会更像面对“一个有当下状态的人”。

### 3. 主动请求系统

除了玩家主动安排行动，也建议加入角色主动提出请求，例如：

- 想吃某样东西
- 想出去走走
- 不想见人
- 想学习一个字

玩家是否响应这些请求，会比普通行动更能塑造关系和角色印象。

### 4. 模糊提示而非完全明牌

建议不要把所有阈值直接展示给玩家，可以通过文本提示给出方向，例如：

- 她今天看起来更愿意交谈
- 她似乎不适合出门
- 她对某个话题表现出兴趣

这样玩家会更像在观察角色，而不只是看数值表。

## 结局建议

建议尽早补上一个基础结局框架，即使先只有 3 到 4 个结局也值得：

- 陪伴结局：高信赖、高心情
- 成长结局：高学识、中高信赖
- 脆弱依赖结局：高信赖但整体状态不健康
- 疏离结局：低信赖或关键事件失败过多

一旦有了结局导向，前面的每天行动才会更有长期意义。

## 最值得优先做的三件事

如果只优先做少量玩法增强，建议顺序如下：

1. 为行动加入负面代价和连续收益递减
2. 增加阶段目标或章节目标
3. 把事件从单点阈值触发升级成事件链和记忆分支

这是当前成本相对可控、但对可玩性提升最明显的三个方向。

## 一句话总结

当前项目下一阶段最值得投入的，不是先扩更多编辑器功能，而是增强“每天为什么要这样选”的决策张力。

玩法一旦有了代价、阶段目标和持续后果，这个原型就会从“能试玩”明显走向“愿意反复玩”。
