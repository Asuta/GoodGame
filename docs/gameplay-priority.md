# GoodGame Editor 玩法优先级

## P0 - 先做，最直接提升可玩性

### 1. 行动代价与收益递减
目标：让玩家每天必须取舍，避免刷单一最优行动。

建议：

- 给高收益行动增加明确副作用
- 连续执行同一行动时降低收益
- 某些行动在特定状态下收益下降或风险上升

原因：

这是当前提升“决策感”最快的改动，成本相对低，回报最高。

### 2. 阶段目标 / 章节目标
目标：解决每天重复推进但缺少中期目标的问题。

建议：

- 每 5 天或 7 天设置一个阶段目标
- 达成阶段目标后解锁新行动、新场景或新事件
- 阶段目标失败时进入不同分支或承受代价

原因：

它能明显增强推进感，让玩家不只是刷数值，而是在追逐目标。

### 3. 事件链与记忆后果
目标：让事件不再是孤立触发，而是形成连续叙事。

建议：

- 事件 A 触发后解锁事件 B
- 某些失败分支保留后续影响
- 后续事件根据历史行为改写台词和结果

原因：

这一步能显著提升剧情厚度和重复游玩价值。

## P1 - 第二阶段做，增强内容层次

### 4. 关系阶段系统
目标：把单一信赖值升级为更有表现力的关系变化。

建议：

- 增加戒备、试探、依赖、主动靠近等阶段
- 不同阶段影响事件、行动和文本风格

原因：

可以把“数值增长”转化成玩家更容易感知的关系演进。

### 5. 状态系统
目标：让每天的体验更有变化，而不是只看长期属性。

建议：

- 增加持续 1 到 3 天的短期状态
- 状态影响行动可用性、收益和事件触发

原因：

这会让系统更像在回应角色当天的真实状态。

### 6. 主动请求系统
目标：增强角色存在感和互动感。

建议：

- 让角色偶尔主动提出需求或请求
- 玩家响应与否影响关系和后续事件

原因：

这类系统非常适合当前题材，能明显增加陪伴感。

## P2 - 后续增强，丰富表现与策略

### 7. 时间段系统
目标：增强生活感和事件分布层次。

建议：

- 把每日行动扩展为早 / 午 / 晚时段
- 不同行动和事件只在特定时段出现

原因：

它能增加节奏变化，但需要改动主循环，适合放在后面。

### 8. 模糊提示系统
目标：减少明牌阈值感，增强观察和代入。

建议：

- 用文本暗示代替一部分数值提示
- 让玩家从反馈中判断角色状态

原因：

这是体验增强项，优先级低于核心循环变化。

### 9. 结局框架扩展
目标：让长期路线和重玩动机更明确。

建议：

- 先做 3 到 4 个基础结局
- 后续再扩展隐藏结局和特殊结局

原因：

结局重要，但最好建立在前面的阶段目标和事件链之上。

## 推荐推进顺序

推荐按下面顺序推进：

1. 行动代价与收益递减
2. 阶段目标 / 章节目标
3. 事件链与记忆后果
4. 关系阶段系统
5. 状态系统
6. 主动请求系统
7. 时间段系统
8. 模糊提示系统
9. 结局框架扩展

## 一句话版本

如果资源有限，先把“行动为什么难选”“这几天到底在追什么”“之前的选择会不会留下后果”这三件事解决掉。
